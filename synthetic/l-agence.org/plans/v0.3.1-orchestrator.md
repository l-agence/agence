## 🔧 Technical Highlights

**Task Queue & Scheduling**:
```typescript
interface TaskContract {
  task_id: string;
  scope: string;              // /workspace/src/lib/
  max_steps: number;          // prevent runaway
  complexity: 'trivial' | 'small' | 'medium' | 'large';
  priority: number;           // from matrix math
  assigned_agent?: string;    // ralph, aider, etc.
  approval_level: 'auto' | 'human_review' | 'security_critical';
}

orchestrator.schedule(tasks) → {
  ralph: [task1, task3],
  aider: [task2],
}
```

**Execution Bridge**:
```bash
# Before agent runs:
bridge.validate({
  scope: /workspace/src/,
  max_steps: 100,
  allowed_ops: ['read', 'write', 'git', 'test'],
  blocked_ops: ['rm -rf', 'git push --force', 'sleep 99999'],
})

# Agent tries dangerous op:
rm -rf /workspace  # ← bridge blocks, kills task
```

**DWM Approval Gate**:
```
Agent completes task-001 → extracts lesson
  "Never auto-heal paths" (from task output)
    ↓
Propose to synthetic/lessons/
  (human sees it, reviews for sanitization)
    ↓
Human approves → merged to shared lessons
  (other agents can read + learn)
    ↓
Lesson tagged with task_id, date, agent
  (full traceability)
```

## 🎓 Key Principles

- **Scope isolation**: One agent per scope (no collisions)
- **Step budgeting**: Prevent infinite loops before they start
- **Human authority**: DWM requires human approval (no auto-publish)
- **Deterministic scheduling**: Matrix math drives allocation (reproducible)

---

## ⚠️ Handoff Notes for Implementation

- Orchestrator is **NOT a coordinator** (no central state)
- It's a **scheduler** (reads matrix, assigns tasks, monitors)
- DWM approval is **manual** (humans decide what's publishable)
- Sessions are **immutable** (append-only for audit)
