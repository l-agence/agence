## 🔐 Key Architectural Decisions

### **1. Path Validation (No Auto-Healing)**

```bash
# BEFORE (VULNERABLE):
validate_path() {
  if ! [[ "$path" =~ ^$ALLOWED ]]; then
    mkdir -p "$(dirname "$path")"    # ← AUTO-HEAL = ESCAPE HATCH
    ln -s ... "$path"                # ← Create junction on demand
  fi
}

# AFTER (SAFE):
validate_path() {
  resolved=$(realpath "$path")       # ← Resolve FIRST
  if ! [[ "$resolved" =~ ^$ALLOWED ]]; then
    reject "Out of scope"            # ← FAIL, don't create
  fi
}
```

**Principle**: Validation ≠ Remediation. Security layer validates only, never creates.

### **2. Symbol Hierarchy (Agent vs. Swarm)**

```
Agent-Level (v0.2.3–v0.3.1, ACTIVE):
  +task           pending
  &task@agent     assigned
  %task@agent     in-progress
  -task           completed
  _task           paused
  #task           held by human

Swarm-Level (v0.3.2+, RESERVED):
  ~task           swarm accepted
  $task           swarm coordinating
```

**Principle**: No scope collision. Prefixes scale hierarchically.

### **3. Scope Model (HERMETIC/NEXUS/SYNTHETIC/ORGANIC)**

```
hermetic/@        never committed (personal notes, todos)
nexus/            local by default, future ailedger (faults, logs, sessions)
synthetic/@       team-shared, Git committed (plans, lessons, issues, docs)
organic/          team work, Git committed (tasks, jobs, workflows)
```

**Principle**: Scope determines sharing + publishing rules.

### **4. Routing (@ Suffix)**

```
task@user         human assigned
task@ralph        agent assigned
task@acme.tld     org-routed (inheritance)
task@acme.tld:team-a  scoped (future swarms)
```

**Principle**: Explicit, composable, extensible.

## 🎓 Lessons Learned

1. **Path-based sandboxing + junction creation = no sandbox** (root cause)
2. **TOCTOU attacks via OS-level redirection** (not string-level validation)
3. **Separation of concerns is critical**: routing (junctions ✓) ≠ security (realpath only)
4. **Future container boundary**: WSL-Ubuntu + Docker = resolved paths are trustworthy
