## 📌 Objective

Implement 2-column tile layout using **tmux** (not Windows Terminal tiling): left (ibash/ishell — human overwatch/hypervisor plane) + right (aibash/aishell — agentic execution, fully observable at all times). Use `script(1)` inside tmux panes for full PTY capture. Enable POSIX job control as coordination primitive.

**Design principle**: Human control pane is an *agentic hypervisor* — at any moment the human can see any work or reasoning in any aibash/aishell pane. No PTY bypass, no direct pipes — governance requires full observability. tmux `pipe-pane` supplements `script(1)` for real-time streaming without replacing it.

## 🔧 Technical Highlights

**Job Control Model**:
```bash
docker run -it container /bin/bash
  ├─ aibash &                  (background, right tile)
  ├─ fg %1                     (bring to foreground if needed)
  ├─ Ctrl+Z → SIGSTOP         (pause aibash)
  └─ Ctrl+K → SIGKILL         (kill aibash, restart if needed)

%jobs output:
[1]   Running     aibash
[2]   Stopped     make test
[3]-  Running     ./deploy.sh
```

**Tile Layout (Mock)**:
```
┌─ Agent: ralph ─────────────────┬─ ralph's aibash ────────────────┐
│ (human can type here)          │ (agent executing tasks)         │
│ > ls -la                       │ $ task-001: optimizing...      │
│ > docker exec ... /bin/bash    │ $ task-001: 45% complete       │
│ >                              │                                 │
├────────────────────────────────┼─────────────────────────────────┤
│ Agent: aider                   │ aider's aishell                 │
│ > Ctrl+K                       │ $ task-002: running tests...   │
│ > kill %aibash (sent)          │ $ task-002: ✓ test_api.sh      │
├────────────────────────────────┼─────────────────────────────────┤
│ Shared Console                 │ Session Metadata                │

```
